## 核心模块

### MCP 网关（mcp-gateway）

MCP Gateway 是系统的核心组件，负责处理所有的 MCP 协议请求：

<CardGroup cols={2}>
  <Card title="请求处理" icon="globe">
    - **统一入口**：`/*` 监听所有 HTTP 请求
    - **路由分发**：根据路由前缀动态分发请求
  </Card>
  <Card title="协议转换" icon="arrows-spin">
    - **协议解析**：解析 JSON-RPC 格式请求
    - **Tool 分发**：解析工具名称，构造调用参数
  </Card>
</CardGroup>

#### 主要功能

<Steps>
  <Step title="请求路由">
    根据路由前缀和后缀（`/sse`、`/message`、`/mcp`）进行请求分发
  </Step>
  <Step title="协议解析">
    对 JSON-RPC 格式进行解析，提取方法与参数
  </Step>
  <Step title="服务调用">
    针对外部服务发起调用并解析结果
  </Step>
  <Step title="配置读取">
    动态加载和应用配置信息
  </Step>
</Steps>

## 数据流

<Steps>
  <Step title="请求接收">
    MCP Client 发送请求到 MCP Gateway 的指定端点
  </Step>
  <Step title="协议解析">
    Gateway 解析 MCP 协议格式，提取工具调用信息
  </Step>
  <Step title="配置查找">
    根据工具名称查找对应的配置信息
  </Step>
  <Step title="服务调用">
    使用配置的模板和参数调用后端服务
  </Step>
  <Step title="响应处理">
    处理后端服务响应，转换为 MCP 协议格式
  </Step>
  <Step title="结果返回">
    将处理结果返回给 MCP Client
  </Step>
</Steps>

## 扩展性设计

MCP Gateway 采用模块化设计，支持以下扩展：

- **协议扩展**：支持新的 MCP 协议版本
- **存储扩展**：支持新的存储后端
- **认证扩展**：支持多种认证机制
- **监控扩展**：集成各种监控和日志系统
